Guide de Démarrage Local

Local Setup & Launch Guide

Suivez ces étapes pour lancer l'application JobLister sur votre serveur de développement local.

Follow these steps to run the JobLister application on your local development server.

Prérequis Système

System Prerequisites

Assurez-vous d'avoir installé les logiciels suivants avant de commencer :

Ensure you have the following software installed before beginning:

PHP
^8.2
Composer
Recommandé
Node.js / NPM
v16+ / NPM v8+
MySQL / MariaDB
Port 3306
1

Cloner le Projet

Clone the Repository

Commencez par récupérer le dépôt Git localement en exécutant les commandes ci-dessous dans votre terminal :

Begin by cloning the Git repository locally by running the commands below in your terminal:

git clone https://github.com/Cherkaoui7/jobing.git
cd jobing
2

Configurer le Fichier .env

Configure the .env File

Dupliquez le fichier exemple pour créer votre propre fichier de configuration locale .env :

Duplicate the example configuration file to create your local .env file:

copy .env.example .env

Note : Sous macOS / Linux, utilisez plutôt : cp .env.example .env Note: On macOS / Linux, use instead: cp .env.example .env

Modifiez ensuite les lignes de configuration de base de données dans votre fichier .env avec vos accès MySQL :

Next, open the .env file and configure your database parameters as shown below:

# Paramètres de base de données
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=job_lister
DB_USERNAME=root
DB_PASSWORD= (Laissez vide si aucun mot de passe)
3

Installer les Dépendances

Install Dependencies

Installez les dépendances PHP (Laravel) et les dépendances JavaScript (React / Vue / BootStrap) :

Install the PHP packages (Laravel dependencies) and JavaScript packages (Vue/React assets):

composer install
npm install
4

Générer la Clé et Compiler les Assets

Generate App Key & Build Assets

Générez la clé de sécurité de l'application et compilez les scripts frontend (JSX/Vue/Sass) :

Generate the secure application encryption key and build the frontend assets:

php artisan key:generate
npm run dev
5

Créer la Base de Données et Migrer

Create Database & Run Migrations

1. Lancez votre serveur MySQL (ex: XAMPP, Laragon, WampServer).
2. Créez une base de données MySQL vide nommée job_lister.
3. Exécutez la commande suivante pour créer toutes les tables et injecter le jeu de données fictives (seeders) :

1. Start your local MySQL service (e.g. XAMPP, Laragon, WampServer).
2. Create an empty database named job_lister.
3. Execute the migration command with the seeding option to create tables and insert fake demonstration data: 

php artisan migrate:fresh --seed
6

Lancer l'Application

Serve the Application

Démarrez le serveur de développement PHP local intégré de Laravel :

Start the Laravel local server to run your site:

php artisan serve
Accès au site : Access the site:
http://127.0.0.1:8000

Dépannage & Erreurs Courantes

Troubleshooting & Common Errors

Erreur lors de "composer install" Error during "composer install"
Si Composer renvoie une erreur liée aux versions de packages, exécutez plutôt : composer update If Composer raises dependency resolution issues, try updating them directly with: composer update
Erreur de connexion à la base de données SQL Database Connection Refused Error
Vérifiez que votre serveur MySQL est bien actif, qu'il tourne sur le port configuré dans le .env (3306 par défaut), et que le nom de la base de données créée correspond bien à DB_DATABASE. Ensure that your local database engine is running, that it uses the correct port defined in the .env (default: 3306), and that you created the empty database with the exact name set in DB_DATABASE.
Modifications frontend non visibles Frontend changes do not appear
Pour recompiler automatiquement les modifications faites sur les composants Vue/React/Sass, exécutez la commande suivante à la place de `dev` : npm run watch. To continuously compile changes in Vue/React components as you save files, use the hot reloading watcher: npm run watch.
